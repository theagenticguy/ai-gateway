## What this tree deliberately omits

Four candidate documents were skipped because their subject does not exist here or the signal would mislead:

- **RPC/tools reference** — the runtime is server-only (Lambda handlers + an agentgateway data plane); there is no RPC/MCP tool server to document. (A CLI reference was originally skipped for the same reason, but a standalone operator CLI does exist under `clients/admin_cli/`, so it is now documented at [reference/cli.md](reference/cli.md).)
- **Ownership**, **risk hotspots**, and **dead-code** analysis — git history is effectively single-author plus bots, and the services are Lambda entry points that nothing imports, so naive authorship, churn, and unreferenced-export signals produce false conclusions rather than useful ones.

## Notable findings surfaced during generation

Three consistency gaps worth engineering attention, each cited in the linked docs:

- **Two disagreeing tier-budget tables.** `team_registration` seeds budgets from FREE/STANDARD/PREMIUM/ENTERPRISE ($10/$1000/$10000/$100000), while `budget_enforcement` runtime defaults use different tier names *and* amounts — sandbox/standard/premium/unlimited ($25/$100/$1000/$10000). See [business-logic](insights/business-logic.md).
- **Admin-created budgets are invisible to enforcement.** `budget_admin` writes budgets under a different DynamoDB key schema than `budget_enforcement` reads. See [contract-map](insights/contract-map.md).
- **Silent IaC waivers.** The CI checkov `skip_check` list hard-suppresses six findings (Lambda DLQ/VPC/concurrency/code-signing) under `soft_fail: false`, absent from the justified `.checkov.yml`. See [tech-debt](insights/tech-debt.md).
